## Erzeugung eines lauffähigen Programms

Für ein lauffähiges Programm muss der Quellcode in die Qt-Entwicklungsumgebung importiert werden und dort dann kompiliert werden. Eine vorkompilierte .exe oder .app gibt es nicht.
Aus diesem Grund ist auch keine Installation und damit einhergehende Konfiguration notwendig. Auch die Deinstallation entfällt. Denn das Programm wird über die Entwicklungsumgebung aufgerufen.

## Beschreibung

NOR kann dazu genutzt werden, um den Gesamtwiderstand beliebiger Netzwerke zu berechnen. Diese Netzwerke können dann  ausgedruckt oder auf einem Speichermedium transportiert werden, um diese dann an einem anderen Endgerät wieder aufzurufen, um sie zu zeigen oder weitere zu bearbeiten.

## Struktur des Codes

Als Leitlinie des Code-Styles wurden aus dem verbindlichen Styleguide (Version 1.8.1) der PHWT entnommen. Als Architektur-Pattern wurde das Model-View-Pattern verwendet.

## Verzeichnisstruktur

Die Lizenznennungen und das Readme liegen auf der obersten Ebene des Repositories. Die Quellcode-Datei befinden sich in dem src-Ordner, in diesem Ordner werden die Klassen nochmals in Model und View getrennt abgelegt. Außerdem liegen dort noch die Bilder bzw. Icons die in unserem Programm verwendet werden.
Im doc-Ordner befinden sich das Klassendiagramm, das Lastenheft sowie die Protokolle.
